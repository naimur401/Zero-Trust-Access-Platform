# Solution
নিচের **conversation + context files** দেখে আসল সমস্যা ২টা লেভেলে:

1) **আপনার `/api/ml/classify-risk` endpoint 500 দিচ্ছে (কোড বাগ)**  
2) আপনি চাচ্ছেন **ML Risk section‑এ “Perfect UI” + সত্যি সত্যি CNN‑LSTM inference রান** করাতে (architecture + implementation দরকার)

আমি দুটোই নিচে **robust** ভাবে দিলাম—corner case সহ। শেষে **২টা alternative approach** ও দিলাম, যদি TFJS/Node‑binding কাজ না করে।

## 1) এখনকার 500 Error – Root cause (context file থেকে)

`app/api/ml/classify-risk/route.ts` এ এই লাইনটা নিশ্চিতভাবে crash করবে:

```ts
behavioralScore: behavioralAnomalies.riskScore,
```

কারণ `lib/zeroTrust.ts` অনুযায়ী `detectBehavioralAnomalies()` **number** রিটার্ন করে (0-100), object না। তাই `.riskScore` undefined → runtime error → 500।

আরেকটা বড় বাগ:

- `catch` ব্লকে `body.userId` ব্যবহার করা হয়েছে, কিন্তু `body` ওই scope‑এ নেই → আরেকটা 500।

আরেকটা API correctness issue:

- error হলে আপনি “mock data” রিটার্ন করছেন কিন্তু `{ status: 500 }` দিচ্ছেন → UI তে fetch `.ok === false` হয়ে error দেখাবে (আপনার `ml-dashboard.tsx` এ তাই হচ্ছে)।

## 2) Robust Fix (Bug‑fix + consistent response shape)

### ✅ Replace: `app/api/ml/classify-risk/route.ts`
এটা এমনভাবে লেখা যাতে:
- JSON invalid হলে 400
- userId missing হলে 400
- MongoDB down হলে inference ঠিকই হবে, শুধু `saved:false` হবে (কিন্তু **200** থাকবে)
- Deep Learning model load/predict fail হলে heuristic fallback হবে
- response shape আপনার `components/ml-dashboard.tsx` এর `MLResult` interface‑এর সাথে match করবে

> নোট: এখানে আমি CNN‑LSTM integration hook করে দিলাম, আপনি নিচের Section‑3 অনুযায়ী model ফাইল add করলেই “real” inference হবে।

```ts
// app/api/ml/classify-risk/route.ts
import type { AccessRequest } from '@/lib/zeroTrust'
import {
  classifyRiskML,
  federatedLearningRiskAssessment,
  detectBehavioralAnomalies,
} from '@/lib/zeroTrust'
import { connectDB } from '@/lib/db'
import { MLResult as MLResultModel } from '@/lib/models'
import { predictRiskCnnLstm } from '@/lib/ml/cnnLstm' // <-- add this file below

export const runtime = 'nodejs'
export const dynamic = 'force-dynamic'

type ClassifyBody = {
  userId?: string
  resourceId?: string
  action?: string
  context?: {
    ipAddress?: string
    deviceId?: string
    location?: string
    userAgent?: string
  }
}

async function readJsonSafe(req: Request): Promise<ClassifyBody | null> {
  const text = await req.text()
  if (!text) return null
  try {
    return JSON.parse(text)
  } catch {
    return null
  }
}

function clamp0to100(n: number) {
  if (!Number.isFinite(n)) return 0
  return Math.max(0, Math.min(100, Math.round(n)))
}

export async function POST(request: Request) {
  const warnings: string[] = []
  let body: ClassifyBody | null = null

  try {
    body = await readJsonSafe(request)
    if (!body) {
      return Response.json(
        { error: 'Invalid or empty JSON body' },
        { status: 400 },
      )
    }

    const userId = String(body.userId || '').trim()
    if (!userId) {
      return Response.json({ error: 'userId is required' }, { status: 400 })
    }

    const accessRequest: AccessRequest = {
      userId,
      resourceId: String(body.resourceId || 'resource-api'),
      action: String(body.action || 'READ'),
      timestamp: Date.now(),
      context: {
        ipAddress: body.context?.ipAddress || '0.0.0.0',
        deviceId: body.context?.deviceId || 'unknown',
        location: body.context?.location || 'Unknown',
        userAgent: body.context?.userAgent || 'unknown',
      },
    }

    // 1) Behavioral + federated (existing logic)
    const behavioralAnomalies = clamp0to100(detectBehavioralAnomalies(accessRequest))
    const federatedRiskScore = clamp0to100(
      federatedLearningRiskAssessment(accessRequest.userId, [accessRequest]),
    )

    // 2) ML score (prefer CNN-LSTM, fallback to heuristic)
    let mlRiskScore: number
    let mlSource: 'cnn-lstm' | 'heuristic' = 'cnn-lstm'
    try {
      mlRiskScore = clamp0to100(await predictRiskCnnLstm(accessRequest))
    } catch (e) {
      mlRiskScore = clamp0to100(classifyRiskML(accessRequest))
      mlSource = 'heuristic'
      warnings.push('CNN-LSTM model unavailable; used heuristic fallback.')
    }

    // 3) Try DB save (non-fatal)
    let saved = false
    let resultId: string | null = null
    try {
      await connectDB()
      const doc = await MLResultModel.create({
        userId: accessRequest.userId,
        resourceId: accessRequest.resourceId,
        mlRiskScore,
        behavioralScore: behavioralAnomalies, // <-- FIX (number)
        federatedScore: federatedRiskScore,
        timestamp: Date.now(),
        features: {
          action: accessRequest.action,
          ipAddress: accessRequest.context.ipAddress,
          location: accessRequest.context.location,
          modelSource: mlSource,
        },
      })
      saved = true
      resultId = String(doc._id)
    } catch (e) {
      // Mongo না থাকলেও UI/ML কাজ করবে
      saved = false
      resultId = null
      warnings.push('Database unavailable; result not persisted.')
    }

    return Response.json({
      mlRiskScore,
      behavioralAnomalies,
      federatedRiskScore,
      analysis: {
        timestamp: new Date().toISOString(),
        userId: accessRequest.userId,
        models: [
          mlSource === 'cnn-lstm' ? 'CNN-LSTM Risk Model' : 'Heuristic ML Risk Classifier',
          'Behavioral Anomaly Detector',
          'Federated Learning Aggregator',
        ],
        confidence: mlSource === 'cnn-lstm' ? 0.93 : 0.88,
      },
      saved,
      resultId,
      warnings,
    })
  } catch (error: unknown) {
    const msg =
      error instanceof Error ? error.message : 'Unexpected error in classify-risk'
    return Response.json(
      { error: msg },
      { status: 500 },
    )
  }
}

export async function GET() {
  try {
    await connectDB()
    const mlResults = await MLResultModel.find().sort({ timestamp: -1 }).limit(50)
    const count = await MLResultModel.countDocuments()
    return Response.json({ mlResults, count })
  } catch (error: unknown) {
    // GET এ 500 না দিয়ে degrade mode (UI-friendly)
    return Response.json(
      { mlResults: [], count: 0, warnings: ['Database unavailable'] },
      { status: 200 },
    )
  }
}
```

## 3) “Real CNN‑LSTM model run” (Deep Learning Integration)

Next.js/TS এ CNN‑LSTM inference চালানোর সবচেয়ে বাস্তবসম্মত উপায়:

### Approach (Recommended)
- **Training offline** (local script দিয়ে)
- **Saved model** → `models/cnn-lstm-risk/model.json` + weights
- API route এ **tfjs-node** দিয়ে model load + predict

### 3.1 Add dependency
```bash
pnpm add @tensorflow/tfjs @tensorflow/tfjs-node
```

> Windows এ `@tensorflow/tfjs-node` ইনস্টল issue করলে নিচের “Alternative‑1/2” দেখুন।

### 3.2 Add: `lib/ml/cnnLstm.ts`
এটা:
- model **একবারই** load করে cache করবে (performance + no reload per request)
- model file না থাকলে **throw** করবে (route তখন heuristic fallback নেবে)

```ts
// lib/ml/cnnLstm.ts
import type { AccessRequest } from '@/lib/zeroTrust'
import path from 'path'

type TF = typeof import('@tensorflow/tfjs-node')

const SEQ_LEN = 10
const FEATURE_DIM = 16

function buildFeatureVector(req: AccessRequest): number[] {
  const hour = new Date(req.timestamp).getHours()
  const offHours = hour >= 22 || hour <= 6 ? 1 : 0
  const unknownLoc = req.context.location === 'Unknown' ? 1 : 0
  const newDevice = req.context.deviceId.startsWith('new-') ? 1 : 0
  const botUA = req.context.userAgent.toLowerCase().includes('bot') ? 1 : 0

  const action = req.action.toUpperCase()
  const aRead = action === 'READ' ? 1 : 0
  const aWrite = action === 'WRITE' ? 1 : 0
  const aDelete = action === 'DELETE' ? 1 : 0

  // cheap hashing to numeric bucket (0..1)
  const hash01 = (s: string) => {
    let h = 0
    for (let i = 0; i < s.length; i++) h = (h * 31 + s.charCodeAt(i)) >>> 0
    return (h % 1000) / 1000
  }

  const ipHash = hash01(req.context.ipAddress || '')
  const deviceHash = hash01(req.context.deviceId || '')
  const resourceHash = hash01(req.resourceId || '')

  const hourNorm = hour / 23

  // total 16 dims (keep consistent with training)
  return [
    hourNorm,
    offHours,
    unknownLoc,
    newDevice,
    botUA,
    aRead,
    aWrite,
    aDelete,
    ipHash,
    deviceHash,
    resourceHash,
    0, 0, 0, 0, 0, // reserved for future signals
  ]
}

function buildSequenceTensor(tf: TF, req: AccessRequest) {
  // Real system হলে এখানে user history থেকে last 10 event feature নেবেন।
  // DB না থাকলে: একই request replicate করে sequence বানাচ্ছি (works, but less accurate).
  const x = Array.from({ length: SEQ_LEN }, () => buildFeatureVector(req))
  // shape: [1, SEQ_LEN, FEATURE_DIM]
  return tf.tensor3d([x], [1, SEQ_LEN, FEATURE_DIM], 'float32')
}

async function getTf(): Promise<TF> {
  return await import('@tensorflow/tfjs-node')
}

declare global {
  // eslint-disable-next-line no-var
  var __cnnLstmModel: import('@tensorflow/tfjs-node').LayersModel | undefined
}

async function loadModelOnce(tf: TF) {
  if (globalThis.__cnnLstmModel) return globalThis.__cnnLstmModel

  const modelPath = path.join(process.cwd(), 'models', 'cnn-lstm-risk', 'model.json')
  const url = `file://${modelPath}`
  const model = await tf.loadLayersModel(url)
  globalThis.__cnnLstmModel = model
  return model
}

export async function predictRiskCnnLstm(req: AccessRequest): Promise<number> {
  const tf = await getTf()
  const model = await loadModelOnce(tf)

  const input = buildSequenceTensor(tf, req)
  const out = model.predict(input) as import('@tensorflow/tfjs-node').Tensor
  const data = await out.data()

  // assume sigmoid output in [0..1]
  const p = Number(data[0] ?? 0)
  input.dispose()
  out.dispose()

  // convert to 0..100
  const score = Math.round(Math.max(0, Math.min(1, p)) * 100)
  return score
}
```

# Alternative Solutions (যদি initial solution কাজ না করে)

## Alternative 1: Client-side TFJS (tfjs-node ছাড়াই)
যদি Windows/Serverless environment এ `@tensorflow/tfjs-node` ঝামেলা করে:
- model `public/models/cnn-lstm-risk/model.json` এ রাখুন
- `ml-dashboard.tsx` এ `@tensorflow/tfjs` দিয়ে browser এ load করে predict করুন
- API শুধু “save result” করবে

Pros: Node native binding লাগবে না  
Cons: model client এ expose হবে, browser performance hit হতে পারে

---

## Alternative 2: Python Microservice (FastAPI + PyTorch/Keras)
True deep learning production‑grade করতে সবচেয়ে stable:
- আলাদা service: `http://localhost:8000/predict`
- Next.js API route ওই service call করবে
- CNN‑LSTM training/inference Python এ clean থাকবে

Pros: Best ML ecosystem, training easy  
Cons: extra service run করতে হবে

---

## Alternative 3: ONNX Runtime (fast + portable)
আপনার CNN‑LSTM model ONNX এ export করে
- `onnxruntime-node` দিয়ে inference

Pros: fast, production friendly  
Cons: export pipeline লাগবে
